Objetivo del Proyecto: Modelos Polinomiales y Regularizados

El objetivo de este proyecto es abordar el ciclo de Machine Learning para construir y evaluar modelos de aprendizaje automtico que permitan predecir la demanda de bicicletas elctricas durante las diferentes estaciones del ao. Adems, se busca identificar los factores ms influyentes en la decisin de los usuarios al rentar una bicicleta elctrica, proporcionando insights valiosos para mejorar el sistema de alquiler y promover la movilidad sostenible.

  • Revisin de duplicados: identificacin y eliminacin de registros duplicados que puedan sesgar los resultados.
  • Anlisis de datos nulos: deteccin de valores faltantes y decisiones sobre su imputacin o eliminacin.
  • Distribuciones de variables: visualizacin y anlisis de la distribucin de las variables, incluyendo la variable objetivo, para detectar sesgos y outliers.
  • Mapas de correlacin: evaluacin de la relacin entre las variables para identificar multicolinealidades y decidir sobre posibles reducciones dimensionales.

Preprocesamiento y Validacion Cruzada: Modelos de Regresin Polinomial y Lasso

Despus de haber organizado los datos en un formato ideal para la construccin de los modelos de Regresin Polinomial Mltiple y Regresin Lasso, procederemos a transformar los datos a una escala comn, definir los hiperparmetros y realizar la validacin cruzada mediante el mtodo Pipeline y GridSearchCV. Esto nos permitir elegir el mejor modelo para cada uno.

Para el modelo de regresin polinomial, aplicaremos varias tcnicas de transformacin y seleccin de modelos:

  • Grado de polinomio: Evaluaremos los grados de polinomio 2 y 3.
  • Normalizacin: Utilizaremos RobustScaler para escalar los datos. Este mtodo elimina la mediana y ajusta los valores segn los cuantiles, siendo robusto frente a outliers.

Por otro lado, para el modelo de regresin Lasso, realizaremos las siguientes transformaciones y selecciones de hiperparmetros:

  • Normalizacin: Utilizaremos MinMaxScaler, el cual escala los valores de las variables al rango 0-1, til para la regularizacin.
  • Seleccin de hiperparmetros: Probaremos diferentes valores del hiperparmetro alpha (que regula la penalizacin L1 del modelo) en el rango de 1 a 5, utilizando validacin cruzada para determinar el mejor valor.
In[]:
# OneHotEncoder para transformar las variables categoricas en numericas. Tambien utilizaremos la clase ColumnTransformer  
preprocessor = ColumnTransformer(
    transformers=[
        ('onehot', OneHotEncoder(drop='first'), ['season', 'weathersit', 'time_of_day', 'weekday']),
        ('scaler', RobustScaler(), ['temp', 'hum', 'windspeed'])
    ],
    remainder='passthrough'  # esto hace que las columnas numericas no sean transformadas, explicarlo en el markdown
)

# pipeline para realizar la transformacion de las variables. One hot enconding (desde el objeto 'preprocessor') y RobustScaler.
pipeline = Pipeline(steps=[
    ('preprocessor', preprocessor),
    ('polynomial_features', PolynomialFeatures()),
    ('regression_model', LinearRegression())
])

# defino el param grid 
param_grid = {'polynomial_features__degree': [2, 3]} 

# se define la estrategia de validacion cruzada con numeros Kfold
kfold = KFold(n_splits=5, shuffle=True, random_state=0)

# se inicializa el objeto GridSearchCV
modelos_grid = GridSearchCV(pipeline, param_grid, cv=kfold, n_jobs=-1, scoring='neg_root_mean_squared_error')

# funcion para evaluar obtener las metricas r2, rmse, mae de los dos modelos y devolver un diccionario
def evaluacion_metricas(y_true, y_pred, modelo_elegido):
    return{
        'Modelo': modelo_elegido,
        'r2': r2_score(y_true, y_pred),
        'RMSE': mean_squared_error(y_true, y_pred, squared=False),
